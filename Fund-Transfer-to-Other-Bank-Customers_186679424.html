NETinfo DBP Version 18.1 : Fund Transfer to Other Bank Customers

Created by Fivi Konstantinou, last modified by 1057-Michael Philippos on 06, 11, 2024

End-users can initiate a money transfer from one of their own accounts (or from an account they have the relevant privileges) to a beneficiary within the bank. End-users can do this by entering the beneficiary account number or selecting a registered beneficiary (which stores the beneficiary account and payment details) or retrieving (most of) a previous fund transfer to the beneficiary from the online transaction status. The payment instruction can be submitted (to the Core System) for execution in real time or at a future date or as a standing order.

As with any other type of Fund Transfer, Transfer to Other Bank Customers conforms to the three steps process (i.e. submit, confirm, complete) and certain configuration needs to be done at the Administrator console for its use.

Product Main Features / Business Requirements

  1. Eligible debit accounts are:

    1. Accounts that fall under a Bank Product that has been configured in NETinfo DBP to allow debits (from its accounts) for the particular transaction type

    2. Accounts that end-user has Input privileges (for the particular transaction type) as defined in User Profile.

    3. Accounts under the Customer selected by end-user from the ‘Owner’ field.

  2. Beneficiary accounts are validated against existing accounts held within the Bank

  3. Fund transfer details can be filled using any of the following methods:

    1. Manual entry

    2. Automatic retrieval of beneficiary and certain payment details from registered beneficiaries

    3. Automatic retrieval of beneficiary and certain payment details from previous fund transfers to the same beneficiary

  4. Cross currency transfers are supported with the option to use a negotiated rate. In a cross currency transfer, the currencies of the Debit Account and Transfer Amount differ.

  5. The Transfer Amount’s eligible currencies in same-currency (i.e. when Transfer Amount currency is the same as the Debit Account currency) and cross currency (i.e. when Transfer Amount currency is different than the Debit Account currency) transfers are configurable through the Transaction Type Currency Admin page; the Transfer Amount currency is validated against the eligible currencies at Submit.

  6. Signatures are supported (configurable).

  7. Calculation of charges is supported (configurable)

  8. SCA functionality is supported (configurable).

  9. Daily limits at customer, account, user and user-account level can be enforced (configurable).

  10. Minimum and Maximum Transaction Limits can be enforced (configurable).

  11. Option to specify different fund transfer descriptions for the sender and beneficiary

  12. Option to schedule the fund transfer for execution now or at a future date, or set as recurring payment. The NETinfo DBP has an embedded payment scheduler; if cannot be provided by Bank, this can be used for the scheduling of future payments and standing order.

  13. Option to attach documents on the Fund Transfer (maximum file size, eligible file types and maximum number of files configurable). The bank must provide the necessary interfaces for retrieving the document, the relevant document repository and anti-virus scanning.

  14. Option to enforce document attachment above a transfer amount threshold

Step 1: Input

The figures below, ‘UI/UX - Step 1: Input - Saved Beneficiary’ and ‘UI/UX - Step: 1 - Once Off Beneficiary’, present the ‘Fund Transfer to Other Bank Customers’ Input Page when using a registered beneficiary or manually entering beneficiary details respectively. Referring to these, end-user is prompted to complete the following mandatory information:

  • From Account (i.e. the debit account)

  • Beneficiary Name and Account

  • Amount to be transferred and associated currency

  • Scheduling (i.e. immediate execution, execution at a future date, recurring execution/standing order); in case of future date end-user is prompted to enter the ‘Execution Date’ while if the Repeated option is selected, end-user is prompted to define the parameters of the recurring payment i.e. Start Date, Final Payment Date or Number of Executions and Frequency

For better customer experience, the above information can be autofilled when:

  • The beneficiary has already been registered and a selection of a ‘From Account' has been made; in this case, end-user selects the ‘Saved’ option in the ‘Beneficiary’ field and subsequently selects one of the registered beneficiaries; the ‘Beneficiary Account, Beneficiary Account Owner, Beneficiary Account Currency, Beneficiary Address, Country, Amount & Currency, From Description and To Description are automatically autofilled as available in the registered beneficiary record.

  • Another Fund Transfer has been made in the past to the same beneficiary (using the ‘Once Off’ option); in this case, end-user selects the ‘Once Off’ option in the ‘Beneficiary’ field and starts typing the name of the beneficiary in the ‘Beneficiary Name field; a list of beneficiaries with names matching the characters entered appears dynamically. User selects the beneficiary and all information from the past transaction are loaded (same fields as autofilled using registered beneficiaries).

End-users can optionally:

  • Enter a Beneficiary Address (available as an option when the ‘Once Off’ selection is made)

  • Register the beneficiary details entered (when the 'Once Off selection is made) by clicking on the associated checkbox

  • Provide a description ('From Description') that will appear in their Online Transaction Status and Transaction History (if returned from the Bank’s Core System).

  • Provide a description (To Description) that will appear in the beneficiary’s transaction details (if returned from the Core System)

  • Use an agreed negotiated rate if the payment amount is in a currency different from the debit account’s currency (end-user has to enter the associated ‘code’).

  • Upload documents to support the Fund Transfer

Once all compulsory information have been entered, end-user proceeds to the next step by pressing the ‘Submit’ button.

A disclaimer may be displayed if configured

Confirm and/or Successful SCA

All validations & checks are successful, and end-user approvals are associated with the transaction.

The following actions are taken by the system:

  • The transaction changes status to ‘Pending for Authorisation’

  • The transaction is registered to the ‘Pending Authorisation’ page

  • The transaction is registered to the ‘Online Transaction Status’ page

  • A ‘Pending for Authorisation’ alert is forwarded to the approvers

  • The ‘Transfer to Other Bank Customers’ alert is forwarded to the inputter

  • The transaction is logged

Confirm and/or Successful SCA

All validations & checks are successful, and transaction has been scheduled for execution at a future date.

The following actions are taken by the system:

  • The transaction changes status to ‘Pending’

  • The ‘Transfer to Other Bank Customers’ alert is forwarded to inputter

  • The transaction is registered to the ‘Online Transaction Status’ page

  • The transaction is logged

Confirm and/or Successful SCA

All validations & checks are successful, and transaction has been scheduled as a Recurring Payment (Standing Order).

The following actions are taken by the system:

  • The Standing Order is created and handled by the Standing Orders module thereafter

  • The transaction appears in the ‘Online Transaction Status’ page

  • The ‘New Standing Order’ alert is forwarded to the inputter

  • The transaction is logged

Confirm and/or Successful SCA

All validations & checks are successful, and transaction will be executed immediately.

The following actions are taken by the system:

  • The transaction is submitted to the Core System for further processing

  • The status changes to ‘Submit’ (or ‘Failed’ if something goes wrong) and then upon response from the Core System to ‘Completed’ or ‘Rejected’

  • The transaction is registered to the ‘Online Transaction Status’

  • The relevant Fund Transfer alert is forwarded to the end-user when transaction changes status to Failed or Completed or Rejected

  • The transaction is logged

  • Export the TX details in PDF

  • View Details

  • Initiate a new transfer

By pressing the ‘Done’ button, end-user is redirected to the Transfers and Payments landing page.

Go to Online Activity

Export Details

End-user selects Export Details.

The transaction details below (same as details displayed in the 'View Details' Page) are exported to PDF; if a field has no value then it is not displayed. The report also displays a Header and the Timestamp upon report creation.

  1. Transaction Status: [Transaction Status]

  2. Reference Number: [Transaction Reference Number as generated by NETinfo DBP]

  3. UTN: [Unique Transaction Number as provided by Core System]; displayed if applicable

  4. From Account: [Account Number]

  5. Beneficiary Account: [Beneficiary Account Number]

  6. Beneficiary Name: [Beneficiary Name]

  7. Beneficiary Alias: [Beneficiary Alias]

  8. Beneficiary Address: [Beneficiary Address]

  9. Beneficiary Country: [Beneficiary Country]

  10. Beneficiary Account Owner: [Masked Account Owner] - retrieved from Core System

  11. Beneficiary Account Currency: [Beneficiary Account Currency] - retrieved from Core System

  12. Amount: [Transfer Amount & CCY]

  13. From Description: [From Description]

  14. To Description: [To Description]

  15. Execution Date: [Execution Date] - Displayed If ‘Now’ or ‘Later’ option is selected

  16. Start Date: [Start Date] - Displayed if ‘Repeated’ option is selected

  17. Final Payment Date: [Final Payment Date OR Calculated value if ‘Number of Executions’] - Displayed if ‘Repeated’ is selected

  18. Number of Executions: [Number of Executions] - Displayed if ‘Repeated’ & 'Number of Executions' is selected

  19. Frequency: [Frequency] - Displayed if ‘Repeated’ is selected

  20. Attachment: [File Name with extension]

  21. Exchange Rate: [Actual Exchange Currency rate to be used] retrieved from Core System (or NETinfo DBP in case of negotiated rate) - Displayed in case of cross currency transaction

  22. Exchange Rate Valid From: [Negotiated rate 'From' Timestamp] - retrieved from NETinfo DBP and displayed in case of Negotiated Rate

  23. Exchange Rate Valid To: [Negotiated rate 'To' Timestamp] - retrieved from NETinfo DBP and displayed in case of Negotiated Rate

  24. Debit Amount: [Amount & CCY to be debited] as retrieved from Core System (or NETinfo DBP in case of negotiated rate)

  25. Charges: [Charges - Amount & CCY] as retrieved from Core System

  26. Approval Details:

    1. Created By: [User Name that has submitted transaction with company name in brackets if available]

    2. Approved By: [User Name that has submitted transaction with company name in brackets if available]

  27. Remaining Signatures: [Approval flow based on Signature Groups] - displayed if signatures apply

Note: Exchange Rate, Exchange Rate Valid From/To, Debit Amount & Charges are not displayed when recurring payment, when later execution and when the transaction is pending authorisation
