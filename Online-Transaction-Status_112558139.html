NETinfo DBP Version 18.1 : Online Transaction Status

Created by Fivi Konstantinou, last modified by 1239-Mythillou Stefani on 19, 11, 2024

End-users can view the status of transactions/requests they have submitted (through the NETinfo DBP digital channels) and/or relate to an account they have view privileges; these may be displayed for selected accounts or customers. Filtering options also exist such as amount, transaction type and status. Typical information displayed is date, transaction details, status and approval details. End-users are also provided with the option to cancel a transaction (in case it has not been submitted for processing) or repeat it (i.e. reload the details of the transaction for resubmission).

Attachments embedded within transactions can also be viewed/downloaded.

Product Main Features / Business Requirements

  1. Displays the status of all transactions (except Mass Payment transactions) submitted by the end-user through the digital channels connected to the NETinfo DPB

  2. Displays the status of transactions to all end-users with view privileges on the associated account

  3. Displays details of the submitted transactions including ‘Remaining Signatories’ in case the transaction is connected to a end-user approval workflow

  4. Ability to cancel a submitted transaction with status ‘Pending’ or ‘Pending for Authorisation’ or ‘Pending Processing’

  5. Ability to reuse/repeat a submitted transaction (feature does not apply for Standing Orders, Goals and Dynamic Page transactions)

  6. The following main transaction states apply:

    1. Pending: The transaction has been scheduled for execution at a later date and the NETinfo DBP Scheduler is used

    2. Submitted: The transaction has been submitted to the Core System but no response received and the timer to get the status is used

    3. Pending for Authorisation: Applies when an approval workflow is connected to the transaction; the transaction is pending approval from an authorised approver

    4. Failed: The transaction could not be submitted to the Core System for a technical reason

    5. Cancelled: The transaction has been cancelled by end-user

    6. Declined: The transaction has been declined by approvers

    7. Pending Execution: The transaction has been submitted and Core System has responded; however processing by Core System has not completed (e.g. could be some kind of manual processing)

    8. Pending Processing: Dynamic Page request has been forwarded (to DBP Admin Console) for internal processing

    9. Under Processing: Dynamic Page request is under processing in DBP Admin Console

    10. Completed: Transaction has been completed in Core System or by NETinfo DBP Admin as applicable

    11. Rejected: Transaction has been rejected in Core System or by NETinfo DBP Admin as applicable

  7. Transactions can be exported in Excel or PDF.

Activity Diagram

The figure below depicts the end-user workflow while on the ‘Online Transaction Status’ page. Referring to this, end-user may search for transactions by selecting the account and period of interest (the option for ‘customer based transactions’ is also available)

 End-user can further filter the transactions fetched through keyword search or using:

  • Amount range

  • Period

  • Currency

  • Transaction Type, and

  • Status

 End-user can perform the following actions on transactions displayed:

  • Cancel any pending transaction (i.e. transaction that has not yet been submitted to Core System for processing or does not require approval) they have initiated

  • Repeat a transaction (i.e. the transaction will be opened with all associated data, edited as required and be allowed to be resubmitted); the feature does not apply for Standing Order, Goals and Dynamic Page transactions

 End-user can also export the displayed transactions to PDF or EXCEL file.

About the Activity Diagram: The Activity Diagram depicts the main activities in the workflow: boxes indicate user activity, decision boxes indicate system decision points and arrow text in brackets indicate major system activity/actions.

Online Transaction Status

The figure below (UI/UX - Online Transaction Status), presents the ‘Online Transaction Status’ Page. Referring to this, end-user may search for the transactions of interest by selecting the required Account (or selecting the ‘Customer based transactions’ option, for cases where the transaction is not linked to an account) and period of interest, and then pressing the ‘Go’ button; system fetches all transactions that match the search criteria.

End-user may further filter the transactions fetched through keyword search or using the filter, which enables filtering by:

  • Amount range

  • Currency

  • Transaction Type

  • Status

Filtering takes effect when pressing the ‘Apply’ button; end-user has also the option to clear the filtering by selecting the ‘Clear’ button

For each transaction displayed in the ‘Transaction Details’ section, the end-user can view the following header information:

  • Created On (i.e. the date of transaction submission/creation)

  • Transaction Type

  • Amount and Currency

  • Status (main statuses are: Pending, Submitted, Pending for Authorisation, Competed, Rejected, Declined, Failed, Pending Execution, Pending Processing and Under Processing)

By clicking on the expand option, additional transaction-related details appear as below:

  • Reference Number (this is the Transaction Reference Number)

  • UTN if applicable (this is the Bank’s transaction reference number)

  • Transaction Description: All mandatory and major optional fields completed by the end-user (i.e. fields with values) when submitting the transaction are displayed; typical information that might be displayed are shown below:

    • From Account

    • To Account

    • Card Number

    • Cardholder Name

    • Card Status

    • Beneficiary Name

    • Beneficiary Bank

    • Debit Amount/Currency

    • Execution Date

    • Charges Paid By

    • Start Date

    • Final Payment Date

    • Frequency

    • From Description

    • To Description

    • Attachment

    • Service Category

    • Service

    • Dynamic Page fields (those configured for display)

  • Additional information, such as the below, are displayed as applicable:

    1. Negotiation Rate: [Negotiated rate] to be applied if applicable

    2. Charges: Charges as applicable when end-user has submitted the transaction

    3. Exchange Rate: Actual Exchange Currency rate to be used [Retrieved from core when end-user has submitted the transaction]

    4. Debit Amount: Actual amount to be debited from end-user’s account [Retrieved from Core System when end-user has submitted the transaction)

  • Approval details (this section is not displayed for the cases where the transaction has been submitted for execution by the NETinfo DBP scheduler/timer):

    • Name of end-user who created the transaction

    • Approvers' name

    • Remaining Signatures

End-user may also perform the following actions:

  • Cancel, any pending transaction they have submitted, by selecting the Cancel option] and then pressing the ‘Submit’ button

  • Repeat a transaction (feature does not apply for Standing Order, Goals and Dynamic Page transactions)

  • Export the displayed transactions to PDF or EXCEL file.
