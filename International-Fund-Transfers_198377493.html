NETinfo DBP Version 18.1 : International Fund Transfers

Created by Fivi Konstantinou, last modified by 1239-Mythillou Stefani on 19, 11, 2024

End-users can initiate an international money transfer (using SWIFT or a clearing code -ABA  or Sort code- to identify Beneficiary Bank) from one of their own accounts (or from an account they have the relevant privileges) to a beneficiary account held within another bank, in another country. End-users can do this by entering the beneficiary and payment details manually or retrieving (most of) them from registered beneficiaries or previous fund transfers to the selected beneficiary. The payment instruction can be submitted (to the Core System) for execution in real time or at a future date or as a standing order.

As with any other type of Fund Transfer, International Fund Transfers conforms to the three steps process (i.e. submit, confirm, complete) and certain configuration needs to be done at the Administrator console for its use.

Product Main Features / Business Requirements

  1. Eligible debit accounts are:

    1. Accounts that fall under a Bank Product that has been configured in NETinfo DBP to allow debits (from its accounts) for the particular transaction type

    2. Accounts that end-user has Input privileges (for the particular transaction type) as defined in User Profile.

    3. Accounts under the Customer selected by end-user from the ‘Owner’ field.

  2. Fund transfer details can be filled using any of the following methods:

    1. Manual entry

    2. Automatic retrieval of beneficiary and certain payment details from registered beneficiaries

    3. Automatic retrieval of beneficiary and certain payment details from previous fund transfers to the same beneficiary

  3. Define beneficiary bank either by BIC or other clearing code (ABA or Sort Code); in case of BIC, end-user can select the bank of interest from a list of banks connected to the SWIFT network

  4. Define intermediary bank either by BIC or other clearing code (ABA or Sort Code); in case of BIC, end-user can select the bank of interest from a list of banks connected to the SWIFT network

  5. Cross currency transfers are supported with the option to use a negotiated rate. In a cross currency transfer, the currencies of the Debit Account and Transfer Amount differ.

  6. The Transfer Amount’s eligible currencies in same-currency (i.e. when Transfer Amount currency is the same as the Debit Account currency) and cross currency (i.e. when Transfer Amount currency is different than the Debit Account currency) transfers are configurable through the Transaction Type Currency Admin page; the Transfer Amount currency is validated against the eligible currencies at Submit.

  7. Signatures are supported (configurable).

  8. Calculation of charges is supported (configurable)

  9. SCA functionality is supported (configurable).

  10. Daily limits at customer, account, user and user-account level can be enforced (configurable).

  11. Minimum and Maximum Transaction Limits can be enforced (configurable).

  12. Option to specify different fund transfer descriptions for the sender and beneficiary

  13. Option to add multiple lines for the purpose of the fund transfer.

  14. Option to schedule the fund transfer for execution now or at a future date, or set as recurring payment. The NETinfo DBP has an embedded payment scheduler; if cannot be provided by Bank, this can be used for the scheduling of future payments and standing order.

  15. Option to attach documents on the Fund Transfer (maximum file size, eligible file types and maximum number of files configurable). The bank must provide the necessary interfaces for retrieving the document, the relevant document repository and anti-virus scanning.

  16. Option to enforce document attachment above a transfer amount threshold

Step 1: Input

The figure below (UI/UX - Step 1: Input - Once Off Beneficiary & UI/UX - Step 1: Input - Saved Beneficiary), presents the ‘International Fund Transfers’ Input Page. Referring to this, end-user is prompted to complete the following mandatory information:

  • From Account (i.e. the debit account)

  • Beneficiary Name and Account

  • Beneficiary Address

  • Search for the Bank based on the BIC/SWIFT Code using the Search BIC page or manually input the Sort/ABA Code

  • Charges paid by: SHA / OUR / BEN

  • Provide a 'Purpose' for the transfer. Multiple lines for purpose can be added (up to 4 lines).

  • Amount to be transferred and associated currency

  • Scheduling (i.e. immediate execution, execution at a future date, recurring execution/standing order); in case of future date end-user is prompted to enter the ‘Execution Date’ while if the Repeated option is selected, end-user is prompted to define the parameters of the recurring payment i.e. Start Date, Final Payment Date or Number of Executions and Frequency

For better customer experience, the above information can be autofilled when:

  • The beneficiary has already been registered and a selection of a ‘From Account' has been made; in this case, end-user selects the ‘Saved’ option in the ‘Beneficiary’ field and subsequently selects one of the registered beneficiaries; the Beneficiary Name, Beneficiary Alias, Beneficiary Address, Country, Beneficiary Account, Bank BIC/SWIFT Code or Sort/ABA Code, Beneficiary Bank Details, Intermediary Bank [28], Intermediary Bank Details, Amount & Currency, Purpose From Description and To Description are automatically autofilled as available in the registered beneficiary record.

  • Another Fund Transfer has been made in the past to the same beneficiary (using the ‘Once Off’ option); in this case, end-user selects the ‘Once Off’ option in the ‘Beneficiary’ field and starts typing the name of the beneficiary in the ‘Beneficiary Name field; a list of beneficiaries with names matching the characters entered appears dynamically. End-user selects the beneficiary and all information from the past transaction are loaded (same fields as autofilled using registered beneficiaries).

End-users can optionally:

  • Add an Intermediary Bank using the Search BIC page

  • Register the beneficiary details entered (when the 'Once Off selection is made) by clicking on the associated checkbox

  • Provide a description ('From Description') that will appear in their Online Transaction Status and Transaction History (if returned from the Core System).

  • Provide a description (To Description) that will appear in the beneficiary’s transaction details (if returned from the Core System)

  • Use an agreed negotiated rate if the payment amount is in a currency different from the debit account’s currency (end-user has to enter the associated ‘code’). Upload documents to support the Fund Transfer

Once all compulsory information have been entered, end-user proceeds to the next step by pressing the ‘Submit’ button.

A disclaimer may be displayed if configured

Confirm and/or Successful SCA

All validations & checks are successful, and end-user approvals are associated with the transaction.

The following actions are taken by the system:

  • The transaction changes status to ‘Pending for Authorisation’

  • The transaction is registered to the ‘Pending Authorisation’ page

  • The transaction is registered to the ‘Online Transaction Status’ page

  • A ‘Pending for Authorisation’ alert is forwarded to the approvers

  • The ‘International Transfer’ alert is forwarded to the inputter

  • The transaction is logged

Confirm and/or Successful SCA

All validations & checks are successful, and transaction has been scheduled for execution at a future date.

The following actions are taken by the system:

  • The transaction changes status to ‘Pending’

  • The ‘International Transfer’ alert is forwarded to inputter

  • The transaction is registered to the ‘Online Transaction Status’ page

  • The transaction is logged

Confirm and/or Successful SCA

All validations & checks are successful, and transaction has been scheduled as a Recurring Payment (Standing Order).

The following actions are taken by the system:

  • The Standing Order is created and handled by the Standing Orders module thereafter

  • The transaction appears in the ‘Online Transaction Status’ page

  • The ‘New Standing Order’ alert is forwarded to the inputter

  • The transaction is logged

Confirm and/or Successful SCA

All validations & checks are successful, and transaction will be executed immediately.

The following actions are taken by the system:

  • The transaction is submitted to the Core System for further processing

  • The status changes to ‘Submit’ (or ‘Failed’ if something goes wrong) and then upon response from the Core System to ‘Completed’ or ‘Rejected’

  • The transaction is registered to the ‘Online Transaction Status’

  • The relevant Fund Transfer alert is forwarded to the end-user when transaction changes status to Failed or Completed or Rejected

  • The transaction is logged
