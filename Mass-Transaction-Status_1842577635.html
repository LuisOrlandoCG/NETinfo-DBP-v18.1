NETinfo DBP Version 18.1 : Mass Transaction Status

Created by 1057-Michael Philippos, last modified on 14, 11, 2024

End-users can view the status of their submitted mass payments (i.e. the mass payment files they have submitted for execution); they may also view the details and status of each individual fund transfer instruction within the mass payment.

Product Main Features / Business Requirements

  1. All mass payment files successfully submitted by end-users (through the digital channels connected to the NETinfo DPB) are displayed to authorised end-users

  2. The following key information on the submitted mass payment is displayed on the main page:

    1. Created On i.e. date on which mass payment has been submitted

    2. File Name i.e. the name of the physical file (containing the payment instructions) that has been submitted

    3. From Account i.e. the debit account

    4. Total Amount i.e. this is the total equivalent amount

    5. Status, which can be:

      1. ‘Pending for Authorisation’, when there are pending approvals to be done

      2. 'Pending', when the approval workflow (if any) has been completed, but submission to Core System is pending on execution date

      3. ‘Processed’, when the payment instructions within the mass payment file have been submitted to Core System for execution

      4. ‘Failed’, when NETinfo DBP could not submit the mass payment file for execution (for some reason)

  3. Ability to:

    1. Retrieve transactions based on number of last transactions & period

    2. Filter retrieved entries using keyword search and by amount

    3. Export Information in EXCEL or PDF format

  4. Ability to view (in a modal) and export (in PDF or EXCEL format) additional details relating to:

    1. The mass payment transaction e.g. status, reference number, mass payment details and processing information

    2. Individual fund transfer transactions with expand/collapse option e.g. transaction details, status, charges & currency conversions

    3. Totals (number of transactions and amounts) on debit account currency equivalent and per currency

  5. Cancelling a non-processed mass payment is not supported

  6. Disclaimer on the page is supported

Activity Diagram

Figure below depicts the workflow of the end-user when accessing the ‘Mass Transaction Status’ page. Referring to this, end-user is first presented with the last 10 mass payments that have been submitted for execution; for each mass payment the following information is displayed:

  1. Created On

  2. File Name

  3. Description

  4. From Account

  5. Total Amount (i.e. total equivalent amount in debit account currency)

  6. Status

End-user has the option to:

  • Change the period of interest (i.e. last 10/20/30 transactions or user defined period)

  • Filter the records retrieved using keyword search and/or by amount

  • Export the transactions retrieved in PDF or EXCEL format

  • View further details on an individual mass payment by ‘clicking’ on the associated record; this opens a modal displaying:

    • Certain mass payment related information, such as reference number, status, template details and processing related information

    • Individual fund transfer transaction details (with expand/collapse option), such as transaction details, status, charges & currency conversions

    • Totals (Number of Transactions and Amounts) on debit account currency equivalent and per currency

    • Option to export in PDF or EXCEL

About the Activity Diagram: The Activity Diagram depicts the main activities in the workflow: boxes indicate end-user activity, decision boxes indicate system decision points and arrow text in brackets indicate major system activity/actions.

Mass Transaction Status: Main Page

The figure below (UI/UX - Mass Transaction Status: Main Page), presents the ‘Mass Transaction Status' main page. Referring to this, when end-user first opens the page, the last 10 mass payment payments are displayed by default; end-user may change this, by selecting the option of interest from the ‘Period’ field (in case of ‘User Defined’ selection, end-user is prompted to enter the ‘From’ and ‘To’ dates) and then pressing the ‘Go’ button.

All mass payments matching the search criteria are displayed in the <Mass Payments> area. End-user can further filter the transactions retrieved through keyword search or by Amount range.

For each mass payment displayed, end-user can view the following information:

  • Created On date: This is the date the mass payment was submitted

  • File Name: This is the name of the physical payment file submitted

  • From Account : This is the debit account, as defined in the ‘Mass Payments’ page

  • Total Amount: Displays the total equivalent amount (in debit account currency)

  • Status: Can be ‘Pending for Authorisation’, ‘Pending’, ‘Processed’ or ‘Failed’

End-user may:

  • Select to see further details of a mass payment by clicking on the associated record - this opens the ‘Mass Payment Details Modal'

  • Export the mass payments displayed by selecting the ‘Export’ option and subsequently selecting the preferred report format (PDF or EXCEL)

A disclaimer may appear at the bottom of the page if configured.

Mass Payment Details Modal

The figure below (UI/UX - Mass Payment Details Modal), presents the modal displayed when end-user selects ‘clicks’ on a mass payment record (from the <Mass Payments> area). Referring to this, end-user may:

  • View certain key information mainly relating to the mass payment, such mass payment details and processing of the payments

  • View the individual fund transfer instructions within the mass payment file and their associated status

  • May expand an individual fund transfer instruction to view additional information

  • View the Totals in debit account equivalent currency, with the option to expand to see the totals per currency

  • Export the information on this page (excluding the information displayed when expanding an individual fund transfer instruction) in PDF or EXCEL format

End-user can close the dialogue box at any time by pressing the ‘Close’ icon

  9. Charges: [Charges -Amount & CCY] as retrieved from Core System - Displayed when transaction is on ‘Completed’ status
