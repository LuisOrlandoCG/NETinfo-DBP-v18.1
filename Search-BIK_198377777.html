NETinfo DBP Version 18.1 : Search BIK

Created by Fivi Konstantinou, last modified by 1246-Spyrou Marios on 08, 10, 2024

In Ruble Fund Transfers, the end-user is required to populate the Beneficiary Bank details (i.e. BIK code and Bank address). To facilitate the process, the list of all available banks connected to the associated network are automatically fetched; end-user is prompted to select the Bank of interest by searching through the list using certain criteria (i.e. City, Bank name, BIK code).

Search BIK modal

The figure below (I/UX - Search BIK), presents the ‘Search BIK’ modal. Referring to this, end-user is prompted to search through the list of available banks connected to the associated network using the following search criteria:

  • City

  • Bank Name

  • BIK Code

Upon selecting ‘Search’, the records matching the search criteria are displayed in the reporting area. By selecting the Bank of interest, the ‘Transfer in Rubles’ input page is populated with the associated Bank details.

Each bank associated with the criteria selected appears as a separate row in the ‘BIK Table’.

The parameters of each bank are shown (in read only format) as columns in the ‘BIK Table’; these are described in the ‘BIK Table’.

End-user can navigate through the pages (forward and backward, to the first and last page) if data do not fit in one page

End-user can define the number of records displayed in a page: Options 10, 25 and 50
